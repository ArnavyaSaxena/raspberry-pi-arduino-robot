# Raspberry Pi – Arduino Robot Platform

## Overview

This project implements a **differential-drive robotic platform** using Raspberry Pi 5 and an Arduino-based microcontroller for motion control and embedded system interfacing.

The Raspberry Pi performs high-level perception and decision-making, while the Arduino is responsible for low-level motor control and actuator interfacing.

Rather than using serial communication between the two controllers, the system employs a **hardware-based command interface**. Directional control signals generated by the Raspberry Pi are passed through a **2-to-4 decoder**, which translates the binary input signals into discrete movement commands that are interpreted by the Arduino to drive the motors accordingly.

---

## Hardware Used

* Raspberry Pi 5
* Arduino (ATmega-based microcontroller)
* Motor Driver
* DC Motors
* 2-to-4 Decoder
* GPS Module
* IMU Sensor

---

## System Architecture

Raspberry Pi → Control Signals → 2:4 Decoder → Arduino → Motor Driver → Motors

The Raspberry Pi generates two digital control signals based on the output of the perception or object detection module. These signals are decoded by the **2-to-4 decoder**, producing four distinct control lines corresponding to robot motion commands:

* Forward
* Stop
* Left
* Right

The decoded outputs are connected to the Arduino, which interprets the control signals and executes the corresponding motor control routines through the motor driver.
